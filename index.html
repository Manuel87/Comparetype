Module Info
What is this site for? First, see what happens when you toggle the »New Typographic Size«. Do you see the difference?! After quite some research and trying several approaches, I came to the conclusion that only a working prototype could have the power to actually change how we size and therefore, perceive type. With this technique, typefaces with similar formal characteristics will actually produce a similar texture in body text. It sounds trivial, but this allows you to spot the actual differences while choosing a typeface for body text without all the hassle of adjusting everything manually. Test it yourself, depending on what fonts you have on your system you could compare, e.g. Calibri and Arial, Thonburi and Helvetica, Georgia and Times or Adobe Garamond and Stempel Garamond. The typographic size is defined by a real graphical dimension of the most popular characters used. In a regular Latin set text these are the lowercase characters, or more precisely the vertical dimension of most lowercase like the »e«. This has to be script specific, e.g. in uppercase set text it is clearly the uppercase size and in other script systems again another corresponding size. Yes, this is not a Latin issue only. With a general transparent introduction of a new typographical size system I hope many things will get clearer in the future, e.g. talking about formal characteristics, choosing and mixing typefaces, analyzing legibility and so on. Another issue I am addressing with this application is the myth of size between digital and print environments. Therefore, either you look on screen or print the site, the font size will be exactly the same! Hope you like it and if you want to know more about please have a look at this corresponding article: [not/yet/online].
